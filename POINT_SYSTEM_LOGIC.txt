# 포인트 적립 시스템 핵심 로직

## 1. 데이터베이스 스키마

### points 테이블
- id: 고유 식별자
- user_id: 사용자 ID (profiles 테이블 참조)
- amount: 포인트 수량 (양수: 적립, 음수: 사용/만료)
- type: 포인트 유형 (earned, used, expired, bonus, refund)
- description: 포인트 적립/사용 사유
- order_id: 관련 주문 ID (선택적)
- expires_at: 만료 날짜
- created_at: 생성 시간

### user_coupons 테이블
- id: 고유 식별자
- user_id: 사용자 ID
- coupon_id: 쿠폰 ID
- is_used: 사용 여부
- used_at: 사용 시간
- used_order_id: 사용된 주문 ID
- expires_at: 만료 시간

## 2. 핵심 비즈니스 로직

### 포인트 적립 조건
- 주문 완료 시: 주문 금액의 1% (최소 100원)
- 리뷰 작성 시: 100포인트
- 회원가입 보너스: 500포인트
- 특별 이벤트: 관리자 설정

### 포인트 사용 규칙
- 최소 사용 단위: 100포인트
- 최소 주문 금액: 10,000원
- 포인트 사용 한도: 주문 금액의 최대 30%
- 만료 기간: 적립 후 1년

### 포인트 만료 처리
- 매일 자정에 만료된 포인트 자동 차감
- 만료 30일 전 알림 발송
- 만료된 포인트는 복구 불가

## 3. 상태 관리 구조 (usePointStore)

### 주요 상태
- balance: 현재 보유 포인트
- transactions: 포인트 거래 내역
- isLoading: 로딩 상태

### 주요 함수
- fetchUserPoints(userId): 사용자 포인트 조회
- addPoints(userId, amount, type, description): 포인트 적립
- usePoints(userId, amount, orderId): 포인트 사용
- getExpiringPoints(userId): 만료 예정 포인트 조회

## 4. API 함수 구조 (points.ts)

### 포인트 조회
- getUserPoints(userId): 전체 포인트 정보
- getPointHistory(userId, limit): 포인트 내역
- getPointBalance(userId): 현재 잔액

### 포인트 적립
- earnOrderPoints(userId, orderId, orderAmount): 주문 완료 포인트
- earnReviewPoints(userId, reviewId): 리뷰 작성 포인트
- addBonusPoints(userId, amount, reason): 보너스 포인트

### 포인트 사용
- usePointsForOrder(userId, amount, orderId): 주문 시 포인트 사용
- validatePointUsage(userId, amount, orderAmount): 포인트 사용 유효성 검증

## 5. 유효성 검증 로직 (usePointsValidation)

### 포인트 사용 가능 여부
- 충분한 포인트 보유
- 만료되지 않은 포인트
- 최소 사용 단위 충족
- 주문 금액 대비 사용 한도 준수

### 포인트 적립 가능 여부
- 활성 계정 상태
- 중복 적립 방지 (같은 주문/리뷰)
- 적립 한도 확인

## 6. 통합 포인트

### CustomerProfile.tsx
- 포인트 잔액 표시
- 포인트 내역 조회
- 포인트 통계 (적립/사용/만료)

### Checkout.tsx
- 포인트 사용 선택
- 포인트 사용량 입력
- 최종 할인 금액 계산

### PaymentSuccess.tsx
- 주문 완료 시 포인트 적립
- 적립된 포인트 안내

### orderStore.ts
- 주문 완료 시 포인트 적립 트리거
- 포인트 사용 내역 기록

## 7. 에러 처리

### 주요 에러 상황
- 포인트 부족
- 만료된 포인트 사용 시도
- 잘못된 포인트 사용량
- 데이터베이스 연결 실패

### 에러 응답
- 사용자 친화적 메시지
- 로그 기록
- 관리자 알림 (필요시)

## 8. 성능 최적화

### 캐싱 전략
- 포인트 잔액 캐싱 (5분)
- 거래 내역 페이지네이션
- 불필요한 API 호출 방지

### 배치 처리
- 만료 포인트 일괄 처리
- 대량 포인트 적립 시 트랜잭션 사용
