📊 User Flow 범례

🔹 노드 유형

S
시작/종료: 사용자 여정의 시작점과 끝점
P
프로세스: 사용자가 수행하는 작업이나 화면
의사결정: 분기점이나 선택 상황
E
결과: 성공, 실패, 오류 상태

🎯 주요 여정

1단계: 앱 실행 및 인증 (Start → Home)
2단계: 점포 선택 (Home → StoreDetail)
3단계: 상품 선택 및 장바구니 (ProductList → Cart)
4단계: 주문 및 결제 (OrderReview → OrderSuccess)
5단계: 주문 추적 및 완료 (TrackOrder → End)

🎭 주요 사용자 시나리오

🏃‍♂️ 신규 사용자 - 첫 주문

  • 앱 다운로드 및 실행
  • 회원가입 및 프로필 설정
  • 위치 권한 허용 및 주변 점포 확인
  • 점포 선택 및 상품 둘러보기
  • 첫 상품 장바구니 추가
  • 픽업 주문 및 카드 결제
  • 주문 추적 및 픽업 완료

💨 기존 사용자 - 빠른 주문

  • 앱 실행 (자동 로그인)
  • 최근 방문 점포 선택
  • 즐겨찾기 상품 빠른 추가
  • 저장된 결제 방법으로 즉시 결제
  • 주문 완료 알림 수신

🚗 배달 주문 시나리오

  • 홈에서 배달 가능 점포 필터링
  • 최소 주문 금액 확인
  • 배달 주소 입력 및 검증
  • 배달비 포함 총액 확인
  • 배달 시간 선택
  • 결제 및 주문 확정
  • 실시간 배달 추적

❌ 오류 상황 처리

  • 점포 운영시간 외 접근
  • 품절 상품 주문 시도
  • 배달 불가 지역 주문
  • 결제 실패 상황
  • 네트워크 연결 오류
  • 주문 취소 요청
