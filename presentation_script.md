# 편의점 통합 솔루션 v2.0 발표 대본

## 1페이지: 제목 슬라이드
안녕하세요. 오늘 발표할 주제는 '편의점 통합 솔루션 v2.0: 실시간 통합 관리 플랫폼'입니다. 

고객, 점주, 본사를 잇는 완결형 디지털 생태계를 구축하는 프로젝트에 대해 소개해드리겠습니다. 저희는 MRI 컴퍼니 팀으로, 이번 프로젝트를 통해 편의점 운영의 디지털 혁신을 제안하고자 합니다.

## 2페이지: 목차
오늘 발표는 총 6개 섹션으로 구성되어 있습니다. 

먼저 프로젝트 개요에서 주제와 선정 배경, 기획 의도, 프로젝트 내용을 설명드리고, 프로젝트 팀 구성 및 역할을 소개하겠습니다. 이어서 프로젝트 수행 절차 및 방법론을 설명한 후, 실제 수행 경과와 성과물을 보여드리겠습니다. 마지막으로 자체 평가 의견과 향후 개선점 및 개인 성장 부분으로 마무리하겠습니다.

## 3페이지: 프로젝트 주제 및 선정 배경
저희 프로젝트는 실시간 데이터 동기화 기반의 편의점 통합 관리 플랫폼을 개발하는 것입니다.

현재 편의점 운영 환경은 주문, 재고, 매출 관리가 파편화되어 있어 비효율이 발생하고 있습니다. 실시간 데이터의 부재로 재고 부족이나 주문 폭주와 같은 현장 상황에 즉각적인 대응이 어렵고, 점주, 본사, 고객 간의 소통 채널이 부족한 실정입니다. 

이러한 문제점을 해결하기 위해 하나의 통합 플랫폼을 통해 모든 관계자가 실시간으로 정보를 공유하고 효율적으로 업무를 처리할 수 있는 솔루션 개발이 필요합니다.

## 4페이지: 프로젝트 기획 의도
저희의 기획 의도는 '고객, 점주, 본사를 잇는 완결형 디지털 생태계 구축'입니다.

본 프로젝트는 편의점 운영의 모든 측면을 하나의 플랫폼에서 처리할 수 있는 통합 솔루션을 제공하고자 합니다. 실시간 데이터 동기화를 통해 비즈니스 민첩성과 효율성을 극대화하는 것이 핵심 목표입니다.

고객에게는 편리한 주문 경험을 제공하고, 점주에게는 매장 운영을 위한 강력한 관리 도구를, 본사에게는 전체 프랜차이즈 네트워크를 효과적으로 모니터링할 수 있는 시스템을 구축합니다.

이를 통해 모든 이해관계자가 상생할 수 있는 디지털 생태계를 조성하여 편의점 비즈니스의 새로운 패러다임을 제시하고자 합니다.

## 5페이지: 프로젝트 내용
프로젝트는 크게 4개 영역으로 구성됩니다.

첫 번째, 고객용 앱입니다. GPS 기반 지점 검색, 상품 주문 및 장바구니, 실시간 주문 추적, 토스페이먼츠 간편 결제, 주문 이력 조회, 출거점기 기능, 푸시 알림 수신, 리뷰 및 평가 기능을 제공합니다.

두 번째, 점주용 대시보드입니다. 실시간 주문 접수 알림, 주문 상태 관리, 스마트 재고 관리, 일/주/월 매출 분석, 본사 물류 요청, 인기 상품 분석, 직원 관리, 프로모션 설정 기능을 포함합니다.

세 번째, 본사용 관리 시스템입니다. 전국 지점 통합 모니터링, 상품 마스터 관리, 물류 승인 및 배송 관리, 전사 데이터 분석, 매출 트렌드 분석, 지점별 성과 비교, 신규 상품 출시 관리, 프로모션 일괄 적용 기능을 구현했습니다.

마지막으로 핵심 통합 기능입니다. Supabase Realtime을 통한 실시간 데이터 동기화, 역할 기반 접근 제어, 자동화된 재고 관리, 클라우드 기반 확장성, 모바일 최적화 UI/UX, 오프라인 작동 지원, 다중 결제 수단 지원, 데이터 백업 및 복구 시스템을 갖추었습니다.

## 6페이지: 활용 장비 및 재료 (기술 스택)
저희가 사용한 기술 스택을 소개하겠습니다.

Frontend 영역에서는 React 19, TypeScript, Vite, Zustand, TanStack Query, Tailwind CSS를 활용했습니다.

Backend & Database는 Supabase, PostgreSQL, Auth, Realtime API, Storage, RLS 정책을 사용했습니다.

외부연동으로는 토스페이먼츠, Naver Maps API, 푸시 알림 서비스, 이메일 서비스를 연동했습니다.

개발/배포 환경으로는 Git, GitHub, Render (PaaS), CI/CD 파이프라인을 구성했습니다.

모든 기술은 최신 버전을 사용하여 안정성과 성능을 보장했으며, 확장성을 고려한 아키텍처로 설계했습니다.

## 7페이지: 프로젝트 구조 (아키텍처)
시스템 아키텍처 구성에 대해 설명드리겠습니다.

저희는 중앙 Supabase DB를 핵심으로 고객, 점주, 본사 애플리케이션이 API와 Realtime 서비스로 연결되는 구조를 구현했습니다.

Frontend는 역할별(customer, store, hq) 모듈식 구조로 분리하여 재사용성을 극대화했습니다. Backend는 Supabase BaaS로 인증, DB, 실시간 통신, 스토리지 등 핵심 기능을 제공합니다.

Database는 17개 정규화된 테이블과 RLS 정책으로 역할 기반 데이터 접근을 제어합니다. 트리거와 함수를 활용한 비즈니스 로직 자동화를 구현했습니다.

## 8페이지: 활용 방안 및 기대 효과
저희 솔루션의 활용 방안과 기대 효과를 세 가지 관점에서 설명드리겠습니다.

첫째, 활용 방안입니다. 중소규모 편의점 프랜차이즈의 즉시 도입이 가능한 통합 솔루션을 제공합니다. 재고 관리 및 자동 발주 시스템이 필요한 소매점과 실시간 주문 처리가 중요한 F&B 매장에 활용할 수 있습니다. 또한 데이터 기반 의사결정이 필요한 소매 비즈니스에 최적화되어 있습니다.

둘째, 운영 효율성 증대입니다. 재고 관리, 주문 처리 자동화로 수작업 감소와 인건비 절감 및 업무 효율성이 향상됩니다. 실시간 데이터 동기화로 업무 지연을 최소화하고, 통합 플랫폼으로 다중 시스템 관리 부담을 해소할 수 있습니다.

셋째, 매출 및 고객 만족도 향상입니다. 데이터 기반 수요 예측으로 품절 최소화가 가능하며, 편매 기회 손실 방지 및 매출 증대 효과를 기대할 수 있습니다. 편리한 주문/결제/수령 기능으로 고객 경험을 개선하고, 실시간 분석 데이터로 신속한 비즈니스 전략을 수립할 수 있습니다.

## 9페이지: 팀 구성 및 역할
프로젝트 성공적인 수행을 위해 각 팀원은 전문 영역에 집중하면서도 긴밀한 협업을 통해 통합된 솔루션을 구축했습니다.

STEP 1의 PM(Project Manager)은 프로젝트 총괄 및 전체 기획을 담당하고, 일정 관리 및 진행 상황을 모니터링했습니다. 요구사항 분석 및 기능 명세를 작성하고, 팀원 간 의사소통을 조율 및 의사결정을 지원했습니다.

STEP 2의 프론트엔드 개발자는 React 19 기반 UI/UX를 개발하고, 역할별(고객/점주/본사) 기능을 구현했습니다. Supabase API 연동 및 상태 관리와 Zustand, TanStack Query를 활용한 데이터 흐름을 관리했습니다.

STEP 3의 백엔드/DB 담당자는 Supabase 데이터베이스를 설계 및 구축하고, RLS 정책 작성 및 보안을 설정했습니다. DB 트리거 및 함수를 구현하고, 실시간 데이터 동기화 로직을 개발했습니다.

프로젝트의 성공적인 수행을 위해 각 팀원은 전문 영역에 집중하면서도 긴밀한 협업을 통해 통합된 솔루션을 구축했습니다. PM은 전체 방향성을 제시하고, 프론트엔드 개발자는 사용자 경험에 초점을 맞추며, 백엔드/DB 담당자는 안정적이고 확장 가능한 데이터 구조를 설계했습니다. 정기적인 코드 리뷰와 스크럼 미팅을 통해 개발 과정에서 발생하는 문제를 신속하게 해결하였습니다.

## 10페이지: 프로젝트 수행 절차 및 방법
저희는 체계적인 4단계 개발 프로세스를 통해 프로젝트를 진행했습니다.

1단계: 기획 및 설계에서는 요구사항 분석 및 기능 명세를 수행하고, Figma를 활용한 와이어프레임과 프로토타입을 제작했습니다. DB 스키마 및 아키텍처를 설계했습니다.

2단계: 개발 과정에서는 Backend-First 접근법을 채택하여 Supabase DB 스키마, RLS와 함수 우선을 개발했습니다. Frontend 개발 및 API를 연동했습니다.

3단계: 통합 및 테스트에서는 전체 기능을 통합하고, 시나리오 기반 테스트를 진행했습니다. 역할별 워크플로우를 검증하고, 피드백을 수집 및 개선했습니다.

4단계: 배포 및 안정화에서는 Render 플랫폼으로 배포하고, 최종 테스트를 진행했습니다. 버그 수정 및 안정화 작업과 성능 최적화를 수행했습니다.

## 11페이지: 수행 경과 1 - 데이터베이스 구축
Supabase 기반 데이터베이스 설계 및 구현 단계에 대해 설명드리겠습니다.

프로젝트의 핵심 기반으로 Supabase와 PostgreSQL을 활용하여 견고한 데이터베이스를 구축했습니다. 특히 원활한 자동 생성 스크립트(supabase-setup/00_setup_all_advanced.sql)를 개발하여 17개 테이블, RLS 정책, 함수, 트리거를 일괄 생성할 수 있게 했습니다.

사용자 역할(고객/점주/본사)에 따라 데이터 접근 권한을 DB 수준에서 제어하는 RLS 정책을 구현하여 보안을 강화했으며, 주문 완료 시 재고 자감 등 비즈니스 로직을 트리거로 자동화했습니다.

## 12페이지: 수행 경과 2 - 핵심 기능 구현
실시간 주문 처리 시스템 구현 및 결제 연동에 대해 설명드리겠습니다.

본 프로젝트의 가장 핵심적인 기능인 실시간 주문 처리 시스템을 성공적으로 구현했습니다. 주문 프로세스는 다음과 같습니다:

1. 고객이 앱에서 상품을 선택하고 토스페이먼츠를 통해 결제를 진행합니다.

2. 결제 성공 시 orders 테이블에 주문 데이터가 생성되며, 이때 DB 트리거가 작동하여 재고를 자동으로 차감합니다.

3. 점주는 Supabase Realtime 구독을 통해 신규 주문 알림을 실시간으로 수신하고, 대시보드가 자동으로 업데이트됩니다.

4. 점주가 주문 상태를 변경하면(예: '처리중'→'준비완료'), 고객 앱에서도 실시간으로 상태 변화를 확인할 수 있습니다.

## 13페이지: 수행 경과 3 - 피드백과 개선
물류 요청 시스템 개선 사례에 대해 설명드리겠습니다.

저희는 개발 과정에서 지속적인 피드백 수집과 개선을 진행했습니다.

초기 문제점으로 재고가 0인 상품이 물류 요청 목록에 나타나지 않아 요청 불가 상황이 발생했습니다.

사용자 피드백을 통해 재고가 없는 상품에 대한 물류 요청의 필요성을 확인했습니다.

개선 내용으로는 INNER JOIN에서 LEFT JOIN으로 쿼리를 수정하여 상품 레코드 없을 시 등록 생성 로직을 추가했습니다.

이를 통해 점주가 필요한 모든 상품에 대해 물류 요청을 할 수 있게 되어 사용성이 크게 개선되었습니다.

## 14페이지: 자체 평가 의견 - 1. 완성도 평가
프로젝트의 완성도를 4개 영역으로 나누어 평가했습니다.

프로젝트 완성도 98%: 기획 단계에서 정의한 모든 핵심 기능이 완전히 구현되었습니다. 주문, 결제, 재고, 분석 등 모든 워크플로우가 완성되었습니다.

기능 구현율 100%: 고객, 점주, 본사 역할별 시나리오 테스트를 통과했으며, 실시간 데이터 동기화와 안정적 작동을 확인했습니다. 대용량 트래픽 테스트를 미실시한 점이 아쉽습니다.

시스템 안정성 95%: 직관적인 UI와 실시간 피드백을 제공하며, 사용자 피드백 반영으로 초기 프로토타입보다 사용성이 개선되었습니다. 역할별 최적화된 인터페이스를 구현했습니다.

사용자 경험 95%: 성능 수준에 근접한 프로토타입을 완성했으며, 기술적으로 모든 기능이 작동하여 실제 비즈니스 환경에서 사용 가능한 수준입니다.

## 15페이지: 자체 평가 의견 - 2. 잘한 점 & 아쉬운 점
프로젝트 수행 과정에서의 강점과 개선점을 분석했습니다.

잘한 점으로는 과감한 최신 기술 도입 및 성공적 적용이 있습니다. React 19, Supabase 등 최신 기술 스택을 적극 활용하여 개발 생산성과 애플리케이션 성능을 동시에 확보했습니다.

BaaS를 활용한 빠른 MVP 개발도 성과입니다. Supabase를 BaaS로 사용하여 백엔드 구축 시간을 획기적으로 단축하고 핵심 비즈니스 로직 구현에 집중할 수 있었습니다.

견고한 아키텍처 설계 측면에서 DB(RLS), API, UI 컴포넌트까지 역할 기반으로 명확하게 분리 설계하여 높은 보안, 확장성, 유지보수성을 확보했습니다.

아쉬운 점으로는 테스트 자동화 부족이 있습니다. 기능 구현 일정에 집중한 나머지, 코드의 안정성을 보장하는 단위 테스트 및 E2E 테스트 코드 작성이 미흡했습니다.

문서화의 깊이 부족도 개선점입니다. 개발 과정의 빠른 속도를 따라잡기 위해 코드 내 주석이나 API 명세 등의 문서화가 일부 부족합니다.

대용량 트래픽 테스트 미실시도 아쉬운 점입니다. 실제 서비스 환경에서의 성능 검증이 이루어지지 않아 확장성에 대한 실질적 검증이 부족했습니다.

## 16페이지: 자체 평가 의견 - 3. 향후 개선점
향후 개선 방향을 4가지 영역으로 제시합니다.

코드 품질 강화: 테스트 커버리지를 확보하고, Vitest, Playwright를 도입하여 테스트 커버리지 80% 이상을 목표로 합니다. CI/CD 파이프라인을 고도화하여 자동 배포 환경을 구축하겠습니다.

성능 최적화: 데이터베이스를 최적화하고, 복잡한 쿼리 실행 계획을 분석하여 필요 컬럼에 인덱스를 추가하겠습니다. 프론트엔드 코드를 스플리팅하여 초기 로딩 시간을 단축하겠습니다.

기능 확장: PWA를 도입하여 푸시 알림 기능을 추가하고, 오프라인 기능을 지원하겠습니다. 고급 데이터 시각화 기능을 구현하여 더욱 직관적인 분석 도구를 제공하겠습니다.

개인 성장: BaaS 활용 능력을 심화하고, 실시간 웹 기능 역량을 증진하겠습니다. 고급 DB 기술을 활용하여 더욱 효율적인 데이터 처리를 구현하고, 문제 해결 능력을 향상시키겠습니다.

## 17페이지: 자체 평가 의견 - 4. 개인 성장
이번 프로젝트를 통해 다양한 측면에서 기술적 성장을 이룰 수 있었습니다.

BaaS 활용 능력 심화: Supabase를 활용한 인증, DB, 실시간 기능을 포함한 플랫폼 애플리케이션을 구축할 수 있는 역량을 확보했습니다.

실시간 웹 기능 역량 증진: WebSocket 기반의 실시간 통신 원리를 이해하고 쇼핑 서비스에 적용하여 뛰어난 사용자 경험을 구현했습니다.

문제 해결 능력 향상: 복잡한 비즈니스 요구사항을 DB 스키마 설계, RLS 정책, 트리거 등 기술적 해결책으로 풀어내는 능력이 향상되었습니다.

빠른 개발과 안정성의 균형: BaaS를 통한 빠른 개발 속도와 함께 테스트와 문서화의 중요성을 체감하며 균형 잡힌 개발 접근법을 배웠습니다.
