화면 ID: STORE-RETURN-001
메뉴 위치: 점주 > 물류 관리 > 반품 요청 탭 > 반품 요청 버튼

화면 개요:
- 점주가 본사에 상품 반품을 요청하는 모달창
- 일반 반품과 물류 요청별 반품 두 가지 유형 지원
- 상품별 상태 정보와 반품 사유를 상세히 입력 가능

UI 구성요소:
| 항목명               | 타입     | 필수 | 설명                                          |
|---------------------|----------|------|-----------------------------------------------|
| 모달 헤더           | Header   | Y    | 제목 "반품 요청", 닫기 버튼                   |
| 반품 유형 선택      | Radio    | Y    | 보유상품 전체 / 물류요청번호별 라디오 버튼    |
| 반품 사유 선택      | Select   | Y    | 7가지 사유(유통기한임박/상품손상 등) + 기타   |
| 기타 사유 입력      | Input    | N    | 반품사유가 '기타'일 때 활성화되는 텍스트입력  |
| 우선순위 선택       | Select   | Y    | 낮음/보통/높음/긴급 4단계 선택                |
| 추가 메모           | Textarea | N    | 추가적인 설명이나 요청사항 자유 입력          |
| 물류 요청 목록      | List     | N    | 물류요청별 반품 시 배송완료된 요청 목록 표시  |
| 물류 요청 카드      | Card     | N    | 요청번호, 배송일, 승인금액, 상품수 정보 표시  |
| 상품 목록 영역      | Section  | Y    | 좌측에 선택 가능한 상품 목록                  |
| 상품 카드           | Card     | Y    | 상품명, 현재재고, 단가 정보와 추가 버튼       |
| 선택된 상품 영역    | Section  | Y    | 우측에 반품할 상품 목록과 상세 정보           |
| 수량 입력           | Number   | Y    | 1부터 현재재고까지 범위의 반품 수량           |
| 상품 상태 입력      | Text     | N    | 손상정도, 유통기한 등 상품 상태 메모          |
| 총 반품 금액        | Summary  | Y    | 선택된 상품들의 총 반품 금액 계산 표시        |
| 로딩 스피너         | Loading  | N    | 데이터 로딩 중 표시되는 스피너                |
| 빈 상태 메시지      | Empty    | N    | 상품이 없을 때 표시되는 안내 메시지           |

버튼 및 액션:
- [닫기 (X)]: 모달 닫기, 모든 입력 데이터 초기화
- [취소]: 모달 닫기, 모든 입력 데이터 초기화
- [반품 요청 제출]: 입력된 데이터로 반품 요청 생성 후 모달 닫기
- [추가]: 상품을 반품 목록에 추가 (중복 선택 시 수량 증가)
- [X (상품 제거)]: 선택된 상품을 반품 목록에서 제거
- 물류 요청 선택: 라디오 버튼으로 특정 물류 요청 선택
- 반품 유형 변경: 라디오 버튼으로 일반/물류요청별 반품 전환

입출력 데이터:
입력
- 반품 유형: 'general' | 'supply_request'
- 반품 사유: 미리 정의된 7가지 사유 또는 사용자 입력 기타 사유
- 우선순위: 'low' | 'normal' | 'high' | 'urgent'
- 추가 메모: 텍스트 (최대 1000자)
- 물류 요청 ID: 물류요청별 반품 시 선택된 요청의 ID
- 상품별 반품 수량: 숫자 (1부터 현재재고까지)
- 상품별 상태 메모: 텍스트 (손상정도, 유통기한 등)

출력
- 반품 요청 생성: return_requests 테이블에 새 레코드 생성
- 반품 아이템 생성: return_request_items 테이블에 상품별 반품 데이터 생성
- 요청번호 자동 생성: 데이터베이스 트리거에서 자동 생성
- 총 반품 금액 자동 계산: 상품 단가 × 반품수량의 합계
- 물류요청 연결: supply_request_id 및 supply_request_item_id 매핑
- 실시간 데이터 갱신: 반품 목록 새로고침 및 성공 콜백 호출ㅍ

특이사항:
- 반품 유형 변경 시 선택된 상품 목록 자동 초기화
- 물류요청별 반품은 배송완료(delivered) 상태의 요청만 선택 가능
- 상품 중복 추가 시 기존 수량에 1 증가 (최대 재고량까지)
- 실시간 총액 계산으로 사용자에게 즉시 피드백 제공
- 각 상품별로 개별적인 상태 메모 입력 가능
