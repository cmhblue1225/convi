화면 ID: STORE-SUPPLY-001
메뉴 위치: 점주 > 물류 관리 > 물류 요청 버튼

화면 개요:
- 점주가 본사에 물류(재고 보충)를 요청하는 모달창
- 상품별 재고 현황을 확인하고 필요 수량을 입력하여 요청 생성

UI 구성요소:
| 항목명               | 타입     | 필수 | 설명                                          |
|---------------------|----------|------|-----------------------------------------------|
| 모달 헤더           | Header   | Y    | 제목 "물류 요청 생성", 닫기 버튼               |
| 우선순위 선택       | Select   | Y    | 낮음/보통/높음/긴급 4단계 선택                |
| 희망 배송일         | Date     | Y    | 날짜 선택기, 오늘 이후 날짜만 선택 가능       |
| 상품 검색창         | Input    | N    | 상품명/단위로 실시간 검색, Ctrl+K 단축키 지원 |
| 검색 결과 요약      | Text     | N    | 검색 키워드와 결과 개수 표시                  |
| 상품 목록 테이블    | Table    | Y    | 전체 상품 목록과 재고 정보 표시               |
| 상품명 컬럼         | Text     | Y    | 상품명과 단위 표시, 검색어 하이라이트        |
| 현재재고 컬럼       | Number   | Y    | 현재 보유 재고 수량                          |
| 안전재고 컬럼       | Number   | Y    | 최소 유지 재고 수량                          |
| 요청수량 입력       | Number   | N    | 0-1000 범위, 요청할 수량 입력                |
| 요청사유 입력       | Text     | N    | 물류 요청 사유 자유 입력                     |
| 비고 입력창         | Textarea | N    | 추가 요청사항 자유 입력                      |
| 빈 상태 메시지      | Section  | N    | 검색 결과 없을 때 표시되는 안내 메시지       |

버튼 및 액션:
- [닫기 (X)]: 모달 닫기, 입력 데이터 초기화
- [취소]: 모달 닫기, 입력 데이터 초기화  
- [요청 생성]: 입력된 데이터로 물류 요청 생성 후 모달 닫기
- [검색 초기화]: 검색어 초기화하여 전체 상품 목록 표시
- [검색창 X 버튼]: 검색어 삭제
- 상품 검색: 실시간 필터링으로 상품명/단위 기준 검색
- Ctrl+K: 검색창에 포커스
- ESC: 검색어 초기화

입출력 데이터:
입력
- 우선순위: 'low'|'normal'|'high'|'urgent'
- 희망 배송일: 날짜 형식 (YYYY-MM-DD)
- 상품별 요청수량: 숫자 (0-1000)
- 상품별 요청사유: 텍스트 (최대 255자)
- 비고: 텍스트 (최대 1000자)
- 검색 키워드: 텍스트 (실시간 필터링)

출력
- 물류 요청 생성: supply_requests 테이블에 새 레코드 생성
- 요청 아이템 생성: supply_request_items 테이블에 상품별 요청 데이터 생성
- 요청번호 자동 생성: "REQ-{timestamp}-{random}" 형식
- 총 요청 금액 자동 계산: 상품 단가 × 요청수량의 합계
- 빈 재고 상품 처리: store_products 테이블에 없는 상품의 경우 새 레코드 생성
- 실시간 데이터 갱신: 요청 목록 새로고침 및 실시간 구독 업데이트
