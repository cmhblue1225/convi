🔧 기술적 구현 세부사항

JWT 토큰 검증

// JWT 검증 로직 const verifyJWT = async (token: string) => { try { const decoded = jwt.verify(token, JWT_SECRET); return { valid: true, payload: decoded }; } catch (error) { if (error.name === 'TokenExpiredError') { return { valid: false, error: 'EXPIRED' }; } return { valid: false, error: 'INVALID' }; } };

알고리즘: HS256 (HMAC-SHA256)

만료시간: 15분 (Access Token), 7일 (Refresh Token)

검증 항목: Signature, Expiry, Issuer, Audience

Supabase Auth 연동

// Supabase 사용자 검증 const validateSupabaseUser = async (userId: string) => { const { data: user, error } = await supabase.auth.admin .getUserById(userId); if (error || !user) { throw new Error('User not found in auth system'); } return user; };

Provider: Supabase Auth

인증 방식: JWT + RLS

세션 관리: Server-side validation

RLS 정책 구현

-- Customer RLS Policy CREATE POLICY customer_access_policy ON orders FOR ALL USING (auth.uid() = customer_id); -- Store Owner RLS Policy CREATE POLICY store_owner_access_policy ON orders FOR ALL USING ( EXISTS ( SELECT 1 FROM stores WHERE stores.id = orders.store_id AND stores.owner_id = auth.uid() ) ); -- Headquarters RLS Policy CREATE POLICY hq_access_policy ON orders FOR ALL USING ( EXISTS ( SELECT 1 FROM profiles WHERE profiles.id = auth.uid() AND profiles.role = 'headquarters' ) );

보안 수준: Row Level Security

적용 범위: 모든 데이터 테이블

성능: Database-level enforcement

보안 강화 기능

// Rate Limiting const rateLimitCheck = async (userId: string, endpoint: string) => { const key = `rate_limit:${userId}:${endpoint}`; const current = await redis.get(key); if (current && parseInt(current) >= RATE_LIMIT) { throw new Error('Rate limit exceeded'); } await redis.incr(key); await redis.expire(key, RATE_LIMIT_WINDOW); }; // Device Fingerprinting const generateFingerprint = (req: Request) => { const components = [ req.headers['user-agent'], req.headers['accept-language'], req.headers['accept-encoding'], req.connection.remoteAddress ]; return crypto .createHash('sha256') .update(components.join('|')) .digest('hex'); };

Rate Limiting: 100 requests/minute per user

IP Filtering: Whitelist/Blacklist 지원

Device Tracking: 의심스러운 로그인 감지

세션 관리

// Session Management interface AuthSession { userId: string; role: 'customer' | 'store_owner' | 'headquarters'; storeId?: string; // store_owner만 permissions: string[]; lastActivity: Date; deviceFingerprint: string; ipAddress: string; } const updateSession = async (session: AuthSession) => { await redis.setex( `session:${session.userId}`, SESSION_TIMEOUT, JSON.stringify(session) ); };

저장소: Redis (In-memory)

만료시간: 30분 비활성 시 만료

동시 세션: 최대 3개 디바이스
